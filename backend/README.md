# Sakshi Clothing Backend Architecture

This is the production-ready backend for Sakshi Clothing.

## Folder Structure

```
backend/
├── src/
│   ├── config/        # Centralized configurations (DB, CORS, JWT, Env)
│   ├── constants/     # Reusable constants (HTTP Status, Error Messages, Enums)
│   ├── controllers/   # Request handlers
│   ├── database/      # Database logic
│   ├── helpers/       # Helper functions
│   ├── logs/          # Log files generated by Winston
│   ├── middlewares/   # Express middlewares (Error handler, Auth, Security)
│   ├── models/        # Mongoose data models
│   ├── repositories/  # Database access layer (Data abstraction)
│   ├── routes/        # API route definitions
│   ├── services/      # Business logic layer
│   ├── uploads/       # File uploads
│   ├── utils/         # Utility functions (ApiError, ApiResponse, Logger)
│   ├── validators/    # Request validation logic
│   ├── app.js         # Express App setup (Middlewares, Route Mounting)
│   └── server.js      # Server entry point
├── .env.example
├── package.json
└── README.md
```

## Architecture

This project follows a **Clean Architecture / layered pattern**:
- **Routes**: Define the API endpoints and map them to controllers.
- **Controllers**: Handle HTTP requests and responses, extracting data to pass to services.
- **Services**: Contain the core business logic.
- **Repositories**: Abstract the database access (Mongoose queries).
- **Models**: Define the database schemas.
- **Middlewares**: Handle cross-cutting concerns like logging, security, and error handling.

## How to Run

1. Clone the repository and navigate to `backend/`.
2. Install dependencies:
   ```bash
   npm install
   ```
3. Copy `.env.example` to `.env` and configure your variables (e.g. `MONGODB_URI`).
4. Start the development server:
   ```bash
   npm run dev
   ```

## Scripts

- `npm run dev`: Starts the server in development mode using `nodemon`.
- `npm start`: Starts the server using Node directly (for production).
- `npm run lint`: Placeholder for linter.
- `npm run format`: Placeholder for formatter.

## Environment Variables

- `PORT`: Server port (default 5000)
- `NODE_ENV`: Application environment (development/production)
- `MONGODB_URI`: MongoDB connection string
- `CLIENT_URL`: Frontend URL for CORS
- `JWT_SECRET`: Secret key for JWT
- `JWT_EXPIRE`: JWT expiration time
- `COOKIE_EXPIRE`: Cookie expiration time
